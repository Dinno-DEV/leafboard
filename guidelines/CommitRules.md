# Feature Documents Commits
1. Commit should only include feature_folder changes or addition.
2. The commit should not include any codebase changes or addition.
3. The commit should not include any asset changes or addition.

# Codebase Commits
1. Commit should only include changes outside of feature_folder and guideline folder.
2. The commit should not tamper feature_folder or guideline folders.
3. The commit can include asset additions, but is discouraged to do it this way.
4. Assets commits and codebase commits should generally be separate commits.

# Typical Commit Structure:
1. feature idea committed
2. feature planned committed
3. assets and resource files committed
4. changes or new code, components, or singleton is committed
5. feature documentation is committed
